# Руководство по стилю иконок

## Общая концепция

Иконки выполнены в деловом, строгом стиле, вдохновлённом интерфейсом 1С:Предприятие 8.2.
Основные принципы:

- **Сдержанная палитра** — приглушённые, деловые тона (steel blue, sage green, muted red/gold)
- **Единый визуальный вес** — все иконки выглядят одинаково "плотно" при 16×16
- **Чёткая геометрия** — прямые линии, аккуратные углы, симметрия где ожидается
- **Минимализм** — никаких лишних деталей, каждый элемент значим
- **Читаемость** — иконка распознаётся при 16×16 px

## Технические параметры

| Параметр | Значение |
|---|---|
| Суперсэмплинг | 4× (рисуем 128×128, уменьшаем до 32×32 и 16×16) |
| Формат | PNG, RGBA, прозрачный фон |
| Рабочая область | координаты 3–28 в 32-пространстве (отступы ~1px при 16×16) |
| Центр | 15.5, 15.5 |
| Библиотека | Pillow (PIL) |
| SCALE | 4 (все координаты умножаются на 4) |

## Правила рисования

### Координаты и масштабирование

```python
R(x1, y1, x2, y2)  # масштабирует bounding box ×SCALE
P([(x,y), ...])    # масштабирует список точек ×SCALE
W(n)                # масштабирует толщину линии ×SCALE
```

### Стандартные толщины линий

| Контекст | Вызов | Пиксели (128px) |
|---|---|---|
| Основной контур фигуры | `W(1.5)` | 6px |
| Тонкие линии (текстуры документов) | `W(0.5)` или `max(1, SCALE//2)` | 2px |
| Жирная линия (акцент, стрелки) | `W(2)` | 8px |
| Очень жирная (галочки, кресты) | `W(3)` | 12px |

### Правила геометрии

1. **Центр** — (15.5, 15.5) для симметричных иконок (круги, стрелки)
2. **Область контента** — от 3 до 28 (отступ 3px в 32-пространстве = ~1.5px при 16×16)
3. **Скругление углов** — `radius=1*SCALE` для прямоугольников, `2*SCALE` для крупных
4. **Симметрия** — навигационные стрелки должны быть зеркальны (prev/next, first/last)
5. **Документы** — всегда с загнутым уголком (dog-ear) справа сверху

### КРИТИЧНО: составные фигуры — единый полигон

Если визуально фигура состоит из нескольких частей, но воспринимается как единое целое —
рисуй её **одним `polygon()`**, а НЕ несколькими примитивами с отдельными outline.

**НЕПРАВИЛЬНО** (видимый шов между фигурами):
```python
# Воронка — треугольник + палочка → видна линия на стыке!
d.polygon(P([(3,4),(28,4),(18,17)]), fill=c, outline=cd, width=W(1.5))
d.rectangle(R(13,17,18,28), fill=c, outline=cd, width=W(1.5))
```

**ПРАВИЛЬНО** (единый контур):
```python
# Воронка — один polygon, контур только по внешнему краю
d.polygon(P([(3,4),(28,4),(18,16),(18,28),(13,28),(13,16)]), fill=c, outline=cd, width=W(1.5))
```

Аналогично: крест (add), Т-образные формы, стрелки с хвостами и т.п.

### КРИТИЧНО: не использовать `line()` для толстых фигур

`line()` с большой толщиной создаёт артефакты на стыках отрезков.
Для галочек, крестиков и подобного — используй `polygon()` с явными точками.

**НЕПРАВИЛЬНО:**
```python
d.line(P([(5,17),(12,25),(27,7)]), fill=PAL['green'], width=W(4))  # виден стык
```

**ПРАВИЛЬНО:**
```python
d.polygon(P([(4,17),(7,13),(12,19),(27,3),(29,6),(12,26)]), fill=PAL['green'], outline=PAL['green_dk'], width=W(1))
```

### Симметрия — вычисляй алгоритмически

Для фигур с повторяющимися элементами (шестерёнка, звезда) —
используй `import math` и цикл по углам. НЕ указывай координаты вручную.

```python
import math
cx, cy = 15.5, 15.5
for i in range(num_teeth):
    angle = math.radians(i * 360 / num_teeth)
    # ... вычислить точки зубца
```

### Думай о целевом размере

Иконка будет отображаться при **16×16 px**. Каждый пиксель на вес золота:
- Если деталь не видна при 16×16 — убирай её, не делай мельче
- Минимальная распознаваемая линия = 1px при 16×16 = W(0.5) при рисовании
- Перед рисованием мысленно представь результат в 16 пикселях

### Правила цвета

1. Каждая фигура имеет fill + outline (тёмный вариант того же цвета)
2. **Синий** — для навигации, действий (стрелки, поиск, фильтр, сохранение)
3. **Зелёный** — для добавления, подтверждения (add, select, галочки)
4. **Красный** — ТОЛЬКО для деструктивных действий (delete, cancel)
5. **Жёлтый/золотой** — для папок и буфера обмена
6. **Серый** — нейтральные элементы (принтер, настройки)
